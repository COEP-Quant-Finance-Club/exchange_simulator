# Project Structure

This document explains the purpose of each major folder in the Exchange Simulator project.

## Folders

- book/
  Handles order book related logic, including storage of buy and sell orders.

- core/
  Contains core data models and shared utilities used across the system.

- engine/
  Responsible for matching logic and trade execution.

- flow/
  Manages order flow, validation, and sequencing of orders.

- ledger/
  Maintains records of executed trades and transactions.

- simulation/
  Used for running end-to-end simulations of the exchange.

- test/
  Contains test cases to verify correctness of the system.
